Proclamations

Special Messages, Proclamations, Military Messages

The Commemorative Messages Admin (CM-Admin) staff person must assign GovCRM roles to Commemorative Messages staff. To be a CM-Admin, your IT administrator must assign you to Active Directory (AD) security group GovOfficeCRMCMAdmin. A list of all GovCRM roles are here.

Message List

The first page is a list of all open messages. (Not CLOSED and not DECLINED.)

➔

Your assignments

  • The first section of cases, Your assignments, will include cases that are assigned to you.
  • If you have role ROUTER, the first section of cases will include New cases.
  • If you have role COORDINATOR, the first section of cases will include cases assigned to the COORDINATOR role.
  • If you have role ROUTER, the first section of cases will include cases assigned to the ROUTER role.

Other work

  • All open cases that are not in the first section will be listed in the second section, Other work. These are the cases that are Open and assigned to other people.

Workflow

The ROUTER selects a New case and decides to Approve or Decline the request.

Buttons

The users with roles Coordinator or Router can see the buttons. Only the ROUTER sees the Approve button.

The Approve and Decline buttons are displayed only if the case status is New.

Approve request

The ROUTER clicks the proc_details_approve_button.png button. A popup asks if you want to send the Approval email. If you click the Cancel button, the action is canceled.

A second popup gives you a second chance to cancel.

If you click the Approve button, an email is automatically sent to the constituent. And these steps occur:

  • Application sets the case status to ➔ Approved
  • Application assigns the case to Role ➔ ROUTER
  • Application adds words to the case activity log
  • An email is automatically sent to the constituent. If there multiple email addresses associated with the constituent, the email is sent to all email addresses. If multiple constituents are associated with the request, the email is also sent to the additional constituents.
  • The email is BCC’d to email distribution group Gov.Messages.Router@hawaiioimt.onmicrosoft.com.
  • The email is BCC’d to you.
  • The Approve and Decline buttons are removed.

If the constituent submits the request 4 weeks early, the following email is sent. The delivery method determines the telephone number in the email body.

Aloha and thank you for contacting the Office of Governor David Ige.

Your request for *message name* has been approved, and our team is working to complete the commemorative message by the due date indicated in your request. We will contact you if we have questions or need clarification.

If you do not receive notice for pick-up within 3 days of your requested due date, please contact our office at 808-586-0034.

Mahalo,
Office of the Governor
State of Hawaiʻi

If the constituent submits a late request, the following email is sent.

Aloha and thank you for contacting the Office of Governor David Ige.

We received your request for *message name* and will do our best to process it by the date indicated. Our team is working to complete commemorative messages on a timely basis, and requests submitted within the timeframe guidelines are given priority.

We will contact you if we have questions or need clarification. In the meantime, please contact our office at 808-586-0034 if you have any questions.

Mahalo,
Office of the Governor
State of Hawaiʻi

Assign to a drafter

The ROUTER clicks the Draft button. The ROUTER then selects a user to write a draft of the message.

  • You may enter an optional Draft due date.
    TODO: Add application enhancement to send an alert email when a case is past due.

  • Users who are assigned role “Commemorative Message Drafter” will appear in the Staff drop down list.

  • If this message is a RUSH, click the Rush checkbox. You must add a rush note. If there is no rush note, the RUSH is ignored. This note will appear at the top of the case details page.

Click the Select button. These steps occur:

  • Application sets the case status to ➔ Drafting.
  • Application assigns the case to the selected User. (developer note: the assigned group is reset to null)
  • Application adds words to the case activity log.
  • An email is automatically sent to the “drafter”.
  • The email is BCC’d to you.
GovCRM Tasker - DRAFTING Job
Kimo, you have been assigned a DRAFTING job. Link to DRAFTING JOB. Due date is Monday October 11, 2021.

If the case is a “RUSH”, the following is sent:

GovCRM Tasker - RUSH Drafting Job RUSH
Kimo, you have been assigned a RUSH drafting job.
Rush note.
Due date is Monday October 11, 2021. Link to DRAFTING JOB.

Drafter uploads document

The drafter clicks the Upload Draft button. The document should be a DOCX file.

  1. If the filename ends with “FINAL”

    • Activity Log
    • Status > Final Uploaded
    • Group > COORDINATOR
    • Attachment Type > MESSAGEFINAL

  2. If the old Status is Drafting OR if the old status is Draft Uploaded

    • Activity Log
    • Status > Draft Uploaded
    • Group > ROUTER
    • Attachment Type > MESSAGE

  3. If the old Status is Editing OR if the old Status is Edit Uploaded

    • Activity Log
    • Status > Edit Uploaded
    • Group > COORDINATOR
    • Attachment Type > MESSAGE

  4. If the old Status is Proofing OR if the old Status is Proof Uploaded

    • Activity Log
    • Status > Proof Uploaded
    • Group > COORDINATOR
    • Attachment Type > MESSAGE

  5. If the old Status is Printing OR if the old status is Print Uploaded

    • Activity Log
    • Status > Proof Uploaded
    • Group > COORDINATOR
    • Attachment Type > MESSAGEFINAL

  6. If none of the above is true, then

    • Activity Log
    • Status > old Status
    • Group > blank
    • Attachment Type > MESSAGE

Upload PDF

  • Activity Log
  • Status > PDF Uploaded
  • Group > COORDINATOR
  • If the file is not a PDF, the document is not saved and an error message is displayed.
