Security

Active Directory Security Groups and Azure App Service Roles

The GovCRM application is secured by Azure Active Directory.

Local on-prem active directory in hawaii.local GOVOU has security groups for this application.

The AD security groups can be assigned to these staff users:

  • Boards and Commissions
  • Casework
  • Commemorative Messages
  • Events with the Governor
  • Flag Requests
  • Meetings with the Governor
  • Admin/Config

A new staff person is added to AD group GovOfficeCRM plus one or more of the other security groups. When the staff person leaves, remove the person from the Active Directory security groups.

Commemorative Messages roles

Commemorative Messages staff also are assigned GovCRM roles. These roles are explained at the Commemorative Messages Roles page.

Active Directory groups are assigned by your IT Administrator. Commemorative Message roles are assigned by your staff Commemorative Messages admin.

When a staffer leaves, keep the username in the database table. The name is used in reports. You may set the user as inactive.

Developer Notes

  1. Create Active Directory groups.
  2. Create Application roles in Azure Active Directory App Registration.
  3. Assign Active Directory groups to Application roles.
  4. Use Application role in C# code.

There is a one-to-one relationship between Active Directory groups and Application roles.

Application roles

Display the list of Application roles by clicking “App roles”.

To create a new Application role, click the Create app role button.

Fill in the Role Display name, Value, and Description. Then click the Apply button.

Assign Active Directory group to Application role

https://docs.microsoft.com/en-us/azure/active-directory/develop/howto-add-app-roles-in-azure-ad-apps

Under Manage, select Users and groups. Select Add user/group to open the Add Assignment pane.

Select Users and groups in the Add Assignment pane. A list of users and security groups is displayed. You can search for GovCRM™ groups by entering “govofficecrm” in the search box. Select the AD group from the list. Click the Select button to proceed.

Select Select a role in the Add assignment pane. All the roles that you’ve defined for the application are displayed. The role descriptions are displayed. Choose a role and click the Select button.

Select the Assign button to finish the assignment of groups to the app.

Confirm that the groups you added appear in the Users and groups list.
